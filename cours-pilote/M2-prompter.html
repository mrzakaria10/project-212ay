Hook

Tu tapes "crée un formulaire React" et tu passes 20 min à retoucher le résultat. Ton collègue obtient exactement ce qu'il veut en un seul prompt. La différence ? La structure de sa demande.

Concept — Les 5 piliers
1. RÔLE
"Tu es expert React senior"
2. CONTEXTE
"Next.js 14 + Tailwind + Prisma"
3. TÂCHE
"Crée un formulaire d'inscription 4 champs"
4. FORMAT
"TypeScript, export default, props typées"
5. CONTRAINTE
"Pas de lib externe, gère loading/error/empty"
Démo — Prompt faible vs fort

Faible : "Crée un formulaire React" → 30 lignes génériques sans validation ni style

Fort : (5 piliers, 10 lignes précises) → 80 lignes prêtes à l'emploi, typées, stylées, avec validation Zod

Action

Choisis un composant de ton projet. Écris un prompt structuré avec les 5 piliers. Compare avec une version sans structure.

HCDA
Hook

La différence entre un prompt faible et un prompt fort peut multiplier ta productivité par 10. C'est la compétence la plus importante du module.

Concept — Match faible vs fort

Prompt faible = vague, pas de contexte → résultat générique. Prompt fort = rôle + contexte + tâche + format + contrainte → résultat exploitable.

Démo — 5 matchs
FAIBLE
"Crée une API"
FORT
"Senior backend, Express + TS + Prisma, CRUD users, JWT"
FAIBLE
"Fais un bouton"
FORT
"Button React + variants + sizes + loading + disabled"
FAIBLE
"Mon code ne marche pas"
FORT
"Erreur + contexte + handler + message exact"
FAIBLE
"Explique les closures"
FORT
"À un junior, analogie vestiaire, 2 exemples"
FAIBLE
"Améliore mon code"
FORT
"Refactore ce composant, sépare en sous-composants, ajoute les types"
Action

Prends un prompt faible récent. Réécris-le en prompt fort avec les 5 piliers. Teste les deux versions. Compare le résultat.
