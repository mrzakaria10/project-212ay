# Leçon 2.2 — Générer du boilerplate utile
## Script Vidéo — ~14 min

### [0:00-0:30] HOOK
> Tu démarres un nouveau projet. Tu passes 15 minutes à créer la structure de dossiers, les fichiers de config, le premier composant, la route de test. Et si tu pouvais tout avoir en 30 secondes ?

### [0:30-3:00] CONCEPT — Le boilerplate, une force de l'IA
> L'IA excelle dans les tâches répétitives et structurées :
> - Structure de projet complète
> - Composants standards (Card, Modal, Button, Layout)
> - Routes CRUD complètes
> - Fichiers de config (ESLint, tsconfig, tailwind)
> - Tests unitaires de base
> 
> Ces tâches sont prévisibles → l'IA les produit parfaitement.

### [3:00-8:00] DÉMO — 3 use cases boilerplate

**Use case 1 — Structure de projet :**
Prompt : 
```
Tu es expert Next.js 14. Génère la structure complète d'un projet
avec : app router, layout, page d'accueil, page about,
composant Header/Footer, configuration Tailwind, dossier lib/,
dossier types/, et un exemple d'API route.
```
→ Résultat : tout le squelette en un prompt.

**Use case 2 — Composant réutilisable :**
```
Tu es expert React + Tailwind. Génère un composant Modal
réutilisable avec : ouverture/fermeture contrôlée par props,
overlay avec fermeture au clic, animation fade, porte via
createPortal, TypeScript, children typé.
```

**Use case 3 — Route CRUD complète :**
```
Génère un fichier de routes Express pour un CRUD de produits.
TypeScript, validation Zod, recherche par nom, pagination.
```

### [8:00-9:00] ACTION
> Choisis un type de boilerplate dont tu as besoin (structure projet, composant, ou CRUD). Génère-le avec un prompt structuré. Compare avec le temps manuel.

### [9:00-9:30] OUTRO
> Dernière leçon : prompts faibles vs prompts forts — le match qu'il te faut pour valider ta technique.
