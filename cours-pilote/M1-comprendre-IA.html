Hook

Tu passes des heures à écrire du boilerplate alors que ton vrai problème, c'est la logique métier ? Tu entends parler de GitHub Copilot, Cursor, Claude, mais tu ne sais pas par lequel commencer.

Concept

3 familles d'outils IA :

  • Assistants IDE (Copilot, Cursor) — complétion en temps réel
  • Chats généralistes (Claude, ChatGPT) — réflexion, architecture
  • Spécialisés (Amazon Q, Replit AI) — écosystèmes spécifiques

Stratégie : maîtrise un assistant IDE + un chat généraliste.

Démo

Copilot : Tape une interface, Copilot propose automatiquement :

interface UserProfile {
  id: string;
  name: string;       // ← Copilot
  email: string;      // ← Copilot
  avatarUrl?: string; // ← optionnel
  createdAt: Date;    // ← typage correct
  isActive: boolean;
}

Puis Claude pour l'architecture API + Cursor pour le debug (Cmd+K).

Action

⏱️ 5 min :

  1. Ouvre VS Code, crée api.ts, tape fetchUserData(id) → observe les suggestions
  2. Va sur Claude.ai : "Explique la différence entre Promise et async/await"

Hook

L'IA peut inventer des fonctions qui n'existent pas, suggérer des failles de sécurité, et vous faire perdre des heures si vous lui faites trop confiance. Cette leçon est la plus importante du module.

Concept — 3 limites
  • Hallucinations — L'IA invente des APIs et méthodes qui n'existent pas
  • Pas de compréhension — Elle prédit statistiquement, ne comprend pas le code
  • Contexte limité — Perd le fil au-delà de ~100K tokens

⚠️ Piège n°1 : le code a l'air correct mais ne l'est pas.

Démo — 3 pièges

Piège 1 — API imaginaire :

await prisma.user.searchVector(query);
// ⚠️ searchVector() n'existe PAS dans Prisma

Piège 2 — Faille sécurité : Path traversal non géré dans un endpoint de download

Piège 3 — Edge case : Division par zéro si tableau vide

Les 5 règles d'or :

  1. Toujours lire le code généré avant de l'exécuter
  2. Toujours tester les cas limites
  3. Ne jamais copier du code que tu ne comprends pas
  4. Vérifier la sécurité
  5. L'IA est un accélérateur, pas un substitut
Action

⏱️ 5 min : Prends un code généré par IA, relis-le ligne par ligne, trouve au moins un problème.
