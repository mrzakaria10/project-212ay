# Leçon 2.3 : Prompts faibles vs prompts forts
## Version Texte HCDA

## H — Hook
La différence entre un prompt faible et un prompt fort peut multiplier ta productivité par 10. C'est la compétence la plus importante du module.

## C — Concept

| | Prompt faible | Prompt fort |
|---|---|---|
| Précision | Vague | Rôle + contexte + tâche |
| Format | Non spécifié | TypeScript, export, props |
| Contrainte | Aucune | Pas de lib, edge cases |
| Résultat | Générique, à retoucher | Quasi exploitable directement |

## D — Démo — 5 matchs

**Match 1 — API :**
- Faible : *"Crée une API"* → 2 routes génériques
- Fort : rôle + stack + CRUD + auth → 10 fichiers prêts

**Match 2 — Composant :**
- Faible : *"Fais un bouton"* → 1 balise button
- Fort : variants + tailles + loading + disabled → composant pro

**Match 3 — Explication :**
- Faible : *"Explique les closures"* → définition Wikipedia
- Fort : analogie concrète + 2 exemples commentés → pédagogique

**Match 4 — Debug :**
- Faible : *"Mon code ne marche pas"* → réponse générique
- Fort : erreur + contexte + code → solution ciblée

**Match 5 — Refacto :**
- Faible : *"Améliore mon code"* → avis vague
- Fort : composant + instructions précises → refacto livrée

## A — Action
Prends un prompt faible que tu as utilisé récemment. Réécris-le en prompt fort. Teste les deux versions. Compare.
